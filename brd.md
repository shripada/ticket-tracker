# Introduction

Ticket tracker app helps users of an organisation to log issues and get them resolved.

# Description

I want to develop a ticketing app that will help
users in any organisation to create tickets and track their
progress and get them resolved. Few users will be admins and
they will be able to assign tickets for resolution, or mark
tickets on hold or close them without resolution too. I want
this as a web app that can be accessed from various devices
including mobile, desktop, laptop etc. A a user, I should be
able to see all tickets I have logged and their status. I should be able to add more information to a in progress ticket as a comment. Admins should be able to see a dashboard as to how many tickets were logged, status, burn rate (may be as a chart), and also be able export a nice report for a duration.

# Additional notes

This app must be a responsive web app
